    Product Ecosystem — DOT.MIND / Dumatel¶

    Карта репозиториев и их связей¶

    Как репозитории связаны между собой¶

    [seadragon] ──REST──► [sprut] ──NATS──► [amigofish]
     │                   │                  │
     │                   ├──NATS──► [anglerfish] ──► [krill]
     │                   │                  │
     │                   ├──HTTP──► [doubloon] ──────────────┐
     │                   │                                   │
     │                   ├──NATS──► [descent]                │
     │                   ├──NATS──► [gull]                   │
     │                   ├──NATS──► [bilge] ◄──[chronograf]  │
     │                   ├──NATS──► [shark] ◄──[chronograf]  │
     │                   └──NATS──► [convictfish] ◄──────────┘
     │
     └──[coral] (UI lib, через Nexus)

Все Python-сервисы используют:
  [orker]  — worker framework
  [psdot]  — shared DB models

    Группировка по назначению¶

    Продукт (основное)¶

    • seadragon — фронтенд (Next.js)
    • sprut — API Gateway (Python)

    Async Workers (Python, все используют orker + NATS)¶

    • amigofish — AI query processing
    • anglerfish — document indexing
    • krill — PDF parsing + OCR
    • descent — Yandex file upload
    • gull — email + Bitrix CRM
    • bilge — subscription billing tasks
    • shark — scheduled tasks
    • convictfish — scheduled tasks

    Standalone Services (FastAPI)¶

    • doubloon — billing (Tinkoff)
    • sealayers — classification
    • kraken — Wolfram Alpha

    Scheduling¶

    • chronograf — task scheduler (pub bilge/shark/convictfish tasks)

    Shared Libraries¶

    • orker — NATS worker framework
    • psdot (sqlalchemy-models-and-migrations) — shared DB models
    • coral — shared UI components

    Frontend Extensions¶

    • react-flow-templates — React Flow pro templates (для Studio?)
    • manta — другой проект (ТОиР, Valamis)
    • nautilus — другой проект (Цифровой двойник, Valamis)

    R&D / ML Research¶

    • langgraph — multi-agent RAG (интеграция в Sprint 23)
    • graphrag — GraphRAG experiments
    • langgraph_evaluation — pipeline evaluation
    • open-ai-agents-list — AI agents research
    • anomaly_detector — anomaly detection
    • file-o-fish — file processing research
    • tabs_agent — agent research
    • rnd — general R&D

    DevOps / Infra¶

    • local-infra — Docker Compose для dev
    • agent_utils — Milvus CLI tools
    • tsunami — load testing (Artillery)
    • static-nginx — static content nginx
    • clown — main nginx config
    • devops/* — CI/CD, environments, infrastructure

    Общие компоненты (psdot models)¶

    Модели из psdot, используемые несколькими сервисами:

    Модель Используется в
    User sprut, amigofish
    Project sprut, anglerfish, amigofish
    Chat sprut, amigofish
    Message sprut, amigofish
    Attachment sprut, anglerfish, krill
    Schedule sprut, chronograf
    Notification sprut, descent
    Organization sprut

    DAL-классы (Data Access Layer): - AttachmentDal — используется в amigofish, anglerfish - ChatDal — amigofish - MessageDal — amigofish - NotificationDal — descent - ScheduleDal — chronograf

    Nexus Package Registry¶

    Корпоративный Nexus используется для публикации внутренних пакетов: - npm: @coral/ui и другие coral packages - pip: orker, psdot (предположительно)

    Клиенты при установке указывают Nexus как дополнительный registry.

    Две отдельные продуктовые линейки¶

    В экосистеме присутствуют два отдельных продукта компании Valamis (возможно, B2G/B2B):

    1. DOT.MIND / Dumatel¶

    • AI-ассистент для B2B-закупок
    • Репозитории: seadragon, sprut, doubloon, workers...

    2. Valamis Platform¶

    • nautilus — "Цифровой двойник" (Digital Twin)
    • manta — "ТОиР" (техническое обслуживание и ремонт)
    • Отдельные фронтенд-репозитории, предположительно используют тот же backend

    Как добавить новый сервис в экосистему¶

    Стандартный паттерн нового Python-воркера: 1. Создать repo с Dockerfile, Makefile, pyproject.toml 2. Подключить orker и psdot как зависимости 3. Определить Task-класс (наследник orker.schemas.tasks.Task) 4. Реализовать async def work(db, task, attempt_id, send_in_progress) 5. Добавить NATS subject в sprut 6. Зарегистрировать в chronograf если нужен scheduler

    Стандартный паттерн нового FastAPI-сервиса: 1. source/src/entrypoints/main.py — FastAPI app + lifespan 2. source/src/api/ — router + routers/ 3. source/src/settings.py — Pydantic Settings 4. source/src/sentry_init.py — Sentry init 5. HTTP client в sprut для интеграции
