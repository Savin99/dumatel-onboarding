    Architecture Deep Dive — DOT.MIND / Dumatel¶

    Архитектурные паттерны¶

    1. Event-Driven / Worker Pattern (NATS JetStream + orker)¶

    Основной механизм асинхронной обработки: sprut публикует задачи в NATS subjects, воркеры (orker-библиотека) консьюмят и обрабатывают их.

    # Паттерн worker (orker library)
async def work(db, task: ParseTask, attempt_id, send_in_progress) -> ParseTaskResult:
    text, path_and_ocrs = await parse_pdf(task.path)
    return ParseTaskResult(text=text, path_and_ocrs=path_and_ocrs)

await worker(work, on_fail=on_fail)

    NATS subjects: - krill_tasks — парсинг PDF - anglerfish_tasks — индексация - amigofish_tasks — AI-запросы - descent_tasks — загрузка в Yandex - gull_tasks — email/CRM - bilge_tasks — биллинг-задачи подписок - shark_tasks — scheduled worker - convictfish_tasks — scheduled worker

    FrontProducer (результаты воркеров → фронтенд через NATS): 

    from orker.utils.nats import FrontProducer
nats_producer = FrontProducer(settings.nats_uri)
await nats_producer.publish(subject, result)

    2. AI Streaming Pattern (NATS → SSE → Browser)¶

    Seadragon (browser)
  → POST /api/chat (Next.js API Route)
    → Sprut REST: инициирует задачу, возвращает stream subject
      → Amigofish воркер: вызывает LLM, стримит события в NATS
        → NATS subjects: `chat.{id}.events`
          → thinker-model.ts: читает NATS JetStream
            → AI SDK 5.0: передаёт события как ReadableStream
              → SSE → браузер → React UI

    Типы событий в NATS stream: - text-delta — инкрементальный текст ответа - reasoning-delta — шаги reasoning AI - tool — вызов инструмента (web search, RAG) - finish — завершение стрима - meta — метаданные

    Resume stream pattern (восстановление после перезагрузки): 

    // seadragon/server/infra/nats/resume-consumer.ts
// DeliverPolicy.StartTime — читает с заданного timestamp
// Позволяет восстановить прерванный стрим после reload страницы

    3. API Gateway Pattern (Sprut)¶

    Sprut — единая точка входа для фронтенда. Аутентифицирует через Keycloak, маршрутизирует к внутренним сервисам:

    API маршруты Sprut: 

    /api/health         — health check
/api/auth           — авторизация (Keycloak)
/api/tree           — файловое дерево
/api/directory      — директории
/api/document       — документы
/api/chat           — чаты
/api/message        — сообщения
/api/project        — проекты
/api/organization   — организации
/api/profile        — профиль пользователя
/api/billing        — биллинг (→ Doubloon)
/api/legal_document — юридические документы
/api/attachment     — вложения
/api/notification   — уведомления
/api/onboarding     — онбординг
/api/yandex         — Yandex интеграция

    Sprut инициализирует при старте: - Keycloak (token introspection) - NATS Producer (publish tasks) - Milvus client (vector search) - HTTP clients: doubloon, bitrix, yandex, yandex_login

    4. Shared Library Pattern (orker + psdot)¶

    orker — унифицированный фреймворк для воркеров: - orker.worker — main loop, retry logic - orker.utils.nats.Producer — NATS publisher - orker.utils.nats.FrontProducer — publisher результатов фронтенду - orker.schemas.tasks.Task — базовый Pydantic класс задачи

    psdot (sqlalchemy-models-and-migrations) — shared models: - SQLAlchemy модели (User, Project, Chat, Message, Attachment, Schedule, etc.) - Alembic миграции - SessionManager, GetDB — управление сессиями - DAL (Data Access Layer): ChatDal, MessageDal, AttachmentDal, ScheduleDal и др.

    5. SPA-like Architecture (Seadragon/Next.js)¶

    Нестандартное использование Next.js App Router: - Route group (workspace) — общий родитель без ремонтирования - LeftArea / StudioArea — клиентские компоненты с условным рендерингом - Адаптеры: route-adapter.ts, LeftAreaAdapter, StudioAreaAdapter - Slot-компоненты без key={chatId} — предотвращение ремонтирования - Архитектура: Feature-Sliced Design (FSD)

    6. Multi-Agent RAG (LangGraph — rag.mind/langgraph)¶

    Модульная система в исследовательской стадии, планируется интеграция в основной продукт: 

    CoordinatorAgent
  classify (JSON LLM) → retrieve → analyze → format_output
      ↓                    ↓          ↓
  use_web/use_kb     RetrievalAgent  AnalystAgent
                     (Tavily+Milvus) (summarize)

    Data Flow — Загрузка и индексация документа¶

    1. Browser → Seadragon → POST /api/attachment → Sprut
2. Sprut → сохраняет Attachment в PostgreSQL (status: pending)
3. Sprut → публикует AttachmentIndexTask в NATS
4. Krill воркер ← NATS
   → парсит PDF (PyPDF + OCR если нужно)
   → возвращает ParseTaskResult в NATS
5. Anglerfish воркер ← ParseTaskResult
   → конвертирует DOCX/PPTX/XLS → PDF (LibreOffice)
   → разбивает на чанки (LlamaIndex)
   → генерирует embeddings (OpenAI)
   → записывает в Milvus
   → обновляет Attachment status: indexed
   → Doubloon hold (списание токенов)
   → FrontProducer: событие completion → NATS → Seadragon
6. Seadragon получает уведомление об успешной индексации

    Data Flow — AI-запрос (Chat)¶

    1. Browser → Seadragon → POST /api/chat/{id}
2. Next.js API Route → Sprut POST /api/message
3. Sprut → сохраняет Message, публикует AmigoFishTask в NATS
4. Amigofish воркер ← NATS
   → billing hold check (Doubloon)
   → вызывает LLM с контекстом чата
   → при RAG запросе: Milvus hybrid search (BM25 + cosine)
   → при веб-поиске: Tavily API
   → стримит события text-delta, reasoning-delta в NATS subjects
5. thinker-model.ts ← NATS JetStream (DeliverPolicy.New)
   → преобразует в AI SDK ReadableStream
   → SSE → браузер
6. При reload: ChatResumeHandler
   → GET /api/chat/{id}/resume?from={timestamp}
   → resume-consumer.ts (DeliverPolicy.StartTime)

    API Контракты¶

    Seadragon ↔ Sprut (OpenAPI)¶

    • OpenAPI spec: https://sprut-api-dev01.monsterscorp.ru/openapi.json
    • Типы генерируются командой: pnpm generate-api-types
    • Файл: seadragon/shared/api/openapi/types.generated.ts

    NATS Event Format¶

    {
  "type": "text-delta" | "reasoning-delta" | "tool" | "finish" | "meta",
  "data": "...",
  "timestamp": "2026-03-12T..."
}

    Task Format (orker)¶

    class AttachmentIndexTask(Task):
    attachment_id: int
    path: str
    type: AttachmentType

class ParseTask(Task):
    path: str

class BilgeTask(Task):
    subscription_id: int
    attempt: int = 1
    billing_period: str = "monthly"

    Аутентификация и авторизация¶

    • Identity Provider: Keycloak
    • Фронтенд: NextAuth.js 5.0-beta.29 (корпоративный SSO)
    • Backend-to-backend: Service user (реализован в Sprint 17)
    • JWT: Keycloak JWT токены, старые refresh_token не инвалидируются (баг DOTMIND-1838)
    • Sprut: при старте await Keycloak().init()

    Deployment¶

    • Cloud: Selectel (OpenStack, регион ru-7)
    • Kubernetes: Selectel MKS, 3–5 нод, 4 vCPU / 16GB RAM каждая
    • GitOps: ArgoCD 8.3.0 — мониторит devops/environments, автосинк
    • Container Registry: cr.selcloud.ru/dotmind/
    • Helm: devops/apps/ — charts для Sprut, Amigofish, Anglerfish; devops/deployer/ — umbrella chart dumatel-system
    • CI/CD pipeline: build → push image → build chart → push chart → update umbrella → deploy (manual)
    • IaC: Terraform (MKS cluster) + Ansible (PostgreSQL, Milvus, MinIO на VM)
    • Monitoring: VictoriaMetrics + Fluent Bit + Sentry
    • Package registry: Nexus (npm packages для coral, pip packages для orker/psdot)
    • Масштабирование: воркеры горизонтально масштабируются через NATS consumer groups; Anglerfish — 2 реплики (index + unindex)

    Потенциальные проблемы (Tech Debt)¶

    1. NextAuth.js beta — версия 5.0-beta.29 (нестабильный API)
    2. Python 3.14 — pre-release версия в продакшне
    3. Keycloak refresh_token — не инвалидируется (открытый баг DOTMIND-1838)
    4. Vitest тесты — незакрытый tech debt (DOTMIND-1749, In Progress)
    5. Coral library — не завершена (задачи DOTMIND-1404/1405 без спринта)
    6. OpenAI token spike — инцидент неконтролируемого списания (решён в Sprint 18)
    7. Coverage 60% — минимальный порог в pytest.ini для Python-сервисов
