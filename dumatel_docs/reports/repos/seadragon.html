    Seadragon -- Фронтенд Dumatel¶

    Назначение и роль в системе¶

    Основное фронтенд-приложение платформы Dumatel. SPA на базе Next.js 15 с App Router. Реализует все пользовательские сценарии: каталог проектов, AI-чат с документами, Studio (инструменты), управление профилем/организацией, биллинг. Является единственным клиентским приложением продукта "Думатель" (Dumatel).

    Seadragon выступает как BFF (Backend-for-Frontend): проксирует запросы к Python-бэкенду Sprut, управляет AI-стримингом через NATS JetStream, и обеспечивает SSR/RSC через Next.js App Router.

    API / интерфейсы¶

    API Routes (Next.js)¶

    Route Method Описание
    /api/chat POST AI-чат: принимает сообщения, публикует complete_tasks в Sprut, слушает NATS JetStream стрим COMPLETIONS, возвращает SSE через AI SDK 5.0
    /api/chat/[chatId]/resume POST Восстановление прерванного AI-стрима (DeliverPolicy.StartTime по timestamps)
    /api/sprut/[...path] ALL Прокси к Sprut API с инъекцией Bearer-токена из NextAuth-сессии
    /api/files/[...path] ALL Прокси к файловому серверу
    /api/health GET { status: 'ok', timestamp }
    /api/dev/nats-stats GET Статистика NATS (только dev-окружение)

    NATS JetStream¶

    • Stream: COMPLETIONS (configurable)
    • Subjects: completion.*
    • Max Age: 5 минут
    • Retry: до 3 NAK-попыток с задержкой 1 секунда
    • Message types: text-delta, reasoning-delta, tool, citations, finish, meta, [DONE]
    • Модуль thinker-model.ts: реализует LanguageModelV2 (Vercel AI SDK) поверх NATS -- позволяет использовать стандартный AI SDK pipeline с кастомным транспортом
    • ai-sdk-bridge.ts: связка между NATS consumer и AI SDK stream
    • resume-consumer.ts: восстановление стрима после обрыва соединения

    AI-стриминг pipeline¶

    Browser -> POST /api/chat
                |
                v
         Sprut (POST /message -> complete_tasks в NATS)
                |
                v
         Amigofish (NATS consumer) -> LLM (OpenAI/etc.)
                |
                v
         NATS JetStream (стрим COMPLETIONS, subject: completion.{chat_id})
                |
                v
         Seadragon server (thinker-model.ts -> NATS consumer -> AI SDK bridge)
                |
                v
         Browser (SSE через AI SDK 5.0 useChat/useChatUI)

    Domain-specific клиенты в shared/api/clients/:

    Клиент Файл Операции
    Profile profile.ts CRUD профиля, фото, пароль
    Chat chat.ts Чаты, сообщения
    Projects projects.ts CRUD проектов
    Documents documents.ts Документы, загрузка, удаление
    Billing billing.ts Тарифы, баланс, транзакции, карты
    Organizations organizations.ts CRUD организаций, участники
    Attachment attachment.ts Аттачменты
    Notification notification.ts Уведомления

    Yandex yandex.ts Yandex Disk OAuth, файлы

    Все клиенты проксируются через /api/sprut/[...path] с автоматической инъекцией Bearer-токена.

    Зависимости от других сервисов¶

    Сервис Протокол Назначение
    Sprut REST (proxy /api/sprut/) Все CRUD-операции, отправка сообщений, управление документами, биллинг
    NATS JetStream nats:// (TCP) AI-стриминг ответов через стрим COMPLETIONS
    Keycloak OIDC (через NextAuth 5.0) Аутентификация, refresh-токены, управление сессией
    File Server HTTP (proxy /api/files/) Загрузка/скачивание файлов, документов
    Sentry HTTPS Error tracking (server + client instrumentation)
    Yandex Metrica JS snippet Аналитика пользователей
    Google Analytics JS snippet (gtag) Аналитика
    Bitrix24 Widget URL Live-чат поддержки
