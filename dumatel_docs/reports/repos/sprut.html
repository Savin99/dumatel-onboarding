    Sprut -- API Gateway / Backend¶

    Назначение и роль в системе¶

    Центральный Python-бэкенд платформы Dumatel. Единая точка входа для фронтенда (Seadragon). FastAPI-приложение, реализующее REST API для всех бизнес-операций: авторизация, управление документами, чатами, проектами, организациями, биллингом, профилем, уведомлениями, интеграцией с Yandex Disk.

    Sprut выполняет роль оркестратора: принимает запросы от фронтенда, работает с PostgreSQL (через SQLAlchemy/asyncpg), управляет векторными коллекциями в Milvus, публикует задачи воркерам через NATS JetStream, проксирует биллинг к Doubloon, интегрируется с Keycloak, Bitrix24 CRM, UniSender и Yandex.

    API / интерфейсы¶

    REST API (17 роутеров, 60+ эндпоинтов)¶

    Аутентификация (/api/auth): - POST /register -- регистрация: создание пользователя в Keycloak + контакт/сделка в Bitrix CRM + подписка в UniSender - POST /verify-email -- верификация email кодом (код хранится в Redis через aiocache) - POST /login -- авторизация через Keycloak, возврат TokenPair (access + refresh) - POST /refresh -- обновление токена - POST /forgot-password -- запрос на сброс пароля (email через SMTP) - POST /reset-password -- сброс пароля

    Документы (/api/document): - POST -- загрузка документа, публикация index_tasks в NATS для индексации - GET / GET /{id} -- список / детали документа - PATCH /{id} -- обновление - DELETE /{id} -- удаление, публикация unindex_tasks в NATS

    Чаты и сообщения (/api/chat, /api/message): - GET /chat -- список чатов пользователя - POST /chat/{id}/stop -- остановка генерации AI-ответа - POST /message -- отправка сообщения: публикация complete_tasks в NATS, сохранение состояния в Redis (complete_{chat_id})

    Проекты (/api/project): полный CRUD Организации (/api/organization): CRUD + управление участниками + реферальный код Профиль (/api/profile): CRUD + фото + смена пароля Биллинг (/api/billing): проксирование к Doubloon -- тарифы, баланс, транзакции, карты, автопродление Аттачменты (/api/attachment): загрузка, получение, удаление файлов; публикация attachment_index_tasks в NATS Нотификации (/api/notification): список уведомлений, пометка прочитанными Онбординг (/api/user/onboarding): welcome popup, реферальная система Yandex (/api/yandex): OAuth авторизация, дерево диска, загрузка файлов с Yandex Disk -> публикация descent_tasks Юридические документы (/api/legal): оферта, политика конфиденциальности, договор (legal_document) Дерево (/api/tree): иерархия директорий в проекте (directory) Здоровье (/api/health): healthcheck

    NATS Subjects (Producer -- публикует задачи)¶

    Subject Payload Потребитель
    index_tasks IndexTask (org_id, doc_id, project_id, metal_path) Anglerfish (индексация документов)
    unindex_tasks UnindexTask (doc_id, project_id) Anglerfish (удаление из индекса)
    complete_tasks CompleteTask (org_id, chat_id, messages, web, mindmap) Amigofish (AI-генерация)
    attachment_index_tasks AttachmentIndexTask (org_id, attachment_id) Anglerfish (индексация аттачментов)
    gull_tasks GullTask (to_email, deal_id) Gull (email-уведомления)
    descent_tasks DescentTask (dir_paths, file_paths, yandex_token) Descent (скачивание с Yandex Disk)

    Аутентификация и авторизация¶

    Для внешних запросов (фронтенд): - JWT через Keycloak JWKS валидация - user_permissions() -- Depends-обёртка, требует валидный Bearer-токен - require_org_access() -- проверка членства в организации - require_project_access() -- проверка доступа к проекту

    Для service-to-service запросов: - users_and_service_header_permissions() -- принимает Bearer-токен ИЛИ заголовок X-Friend-Token (для внутренних вызовов от Amigofish и др.)

    Middleware-стек¶

    1. request_middleware -- логирование запросов (debug mode), X-Request-ID propagation к Sentry
    2. catch_exceptions_middleware -- централизованная обработка ошибок (AuthError, DBError, RequestValidationError, HTTPException, generic Exception)
    3. Exception handlers для WebSocket и HTTP

    Зависимости от других сервисов¶

    Сервис Протокол Направление Назначение
    PostgreSQL asyncpg (TCP) Sprut -> PG Основная БД: пользователи, документы, чаты, проекты, организации (модели из psdot)
    Milvus gRPC (pymilvus) Sprut -> Milvus Создание/удаление векторных коллекций, поиск
    Redis TCP (aiocache) Sprut -> Redis Кеширование: коды верификации email, состояние генерации (complete_{chat_id})
    NATS JetStream nats:// (orker) Sprut -> NATS Публикация задач воркерам (6 subjects)
    Keycloak HTTP Sprut -> Keycloak Создание пользователей, JWT-валидация через JWKS, авторизация
    Doubloon HTTP (internal) Sprut -> Doubloon Биллинг: тарифы, баланс, транзакции, карты, автопродление
    Bitrix24 HTTP webhook Sprut -> Bitrix CRM: создание контактов и сделок при регистрации
    UniSender HTTP Sprut -> UniSender Email-маркетинг: подписка новых пользователей
    Yandex Cloud HTTP Sprut -> Yandex OAuth, Disk API: дерево, скачивание файлов
    SMTP aiosmtplib Sprut -> SMTP Транзакционные email: верификация, сброс пароля
    Sentry HTTPS Sprut -> Sentry Error tracking и performance monitoring
    Seadragon -- Seadragon -> Sprut Фронтенд вызывает Sprut через /api/sprut/ proxy
    Amigofish NATS Sprut (producer) -> Amigofish (consumer) AI-генерация ответов
    Anglerfish NATS Sprut (producer) -> Anglerfish (consumer) Индексация документов
    Gull NATS Sprut (producer) -> Gull (consumer) Email-уведомления
    Descent NATS Sprut (producer) -> Descent (consumer) Загрузка файлов с Yandex Disk
