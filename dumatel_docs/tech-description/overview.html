    Dumatel (DOT.MIND) -- Техническое описание системы¶

    Что такое Dumatel¶

    Dumatel -- SaaS-платформа (B2B) для работы с документами с помощью AI. Система позволяет загружать документы (PDF, DOCX, XLSX, PPTX, изображения, сканы), индексировать их в векторной БД и вести AI-диалог по содержимому. Основные сценарии: RAG-поиск по документам, генерация артефактов (сравнительные таблицы, отчёты, mindmap), веб-поиск с верификацией источников, математические вычисления. Текущий статус: Beta.

    Dumatel развивается как платформа для специалистов, работающих с большим объёмом документации: аналитиков, закупщиков, юристов, исследователей. Монетизация -- подписочная модель с тарифами (Free / PRO / Enterprise) и кредитной системой, оплата через Tinkoff T-Kassa.

    Архитектурный стиль¶

    Система построена на event-driven микросервисной архитектуре с NATS JetStream как центральной шиной сообщений. Ключевые принципы:

    • API Gateway (Sprut) -- единая точка входа для фронтенда, маршрутизирует запросы и публикует задачи в NATS
    • Worker Pattern (orker) -- все асинхронные операции (AI-генерация, индексация, PDF-парсинг, email, биллинг) выполняются NATS-воркерами через единый фреймворк orker
    • AI Streaming -- AI-ответы стримятся через NATS JetStream -> Next.js API Route -> SSE -> браузер с поддержкой resume (восстановление стрима после перезагрузки)
    • Shared Libraries -- единая модель данных (psdot/SQLAlchemy) и единый воркер-фреймворк (orker) для всех Python-сервисов

    Ключевые архитектурные решения¶

    Решение Обоснование
    NATS JetStream как единая шина Persistent messaging для task queue и AI-стриминга. Поддержка consumer groups для горизонтального масштабирования воркеров. Resume stream через DeliverPolicy.StartTime
    FastAPI + asyncpg (Python) Асинхронный стек для I/O-bound нагрузки. Все Python-сервисы используют async
    Next.js 15 как BFF App Router + Server Components + API Routes. Проксирует REST к Sprut, управляет NATS-стримингом AI, обеспечивает SSR
    Milvus (hybrid search) BM25 + dense vectors + RRF для качественного RAG-поиска. Отдельная коллекция на каждый проект
    Keycloak для Identity SSO, управление пользователями, JWT-токены. Интеграция через NextAuth на фронтенде и python-keycloak на бэкенде
    orker + psdot как shared-фундамент Единый воркер-фреймворк и единая модель данных предотвращают дрифт между сервисами
    Hold/Commit/Release (биллинг) Двухфазный учёт кредитов: резервирование перед операцией, списание после завершения. Защищает от перерасхода

    Ограничения и trade-offs¶

    • Python 3.14 -- pre-release версия используется в продакшене (стабильный релиз ожидается октябрь 2026)
    • NextAuth 5.0-beta.29 -- beta-версия в продакшене, нестабильный API
    • Нет RBAC -- текущая модель прав: владелец организации / участник. Гранулярные роли в roadmap (Q1 2027)
    • Polling вместо WebSocket -- часть обновлений статусов всё ещё работает через polling; миграция на сокеты отнесена к техдолгу
    • Coverage 30-70% -- минимальные пороги тестов варьируются между сервисами (30% у psdot, 60% у Sprut, 70% у Doubloon)
    • GPU-зависимость -- Krill (PDF parser) и Anglerfish (marker-pdf) требуют GPU для OCR
    • Один API Gateway -- Sprut является единой точкой отказа; горизонтальное масштабирование через Kubernetes replicas
    • LangGraph -- следующее поколение AI-движка находится в стадии интеграции (Sprint 23), пока параллельно работает legacy Amigofish
