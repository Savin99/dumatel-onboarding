    Dumatel -- Онбординг разработчика¶

    Предварительные требования¶

    Категория Что нужно
    Доступы Почта, GitLab (gitlab.integra-4d.com), YouTrack (youtrack.monsterscorp.ru), Sentry, Kubernetes, БД
    Инструменты Docker, Docker Compose, Git, Node.js 22+, pnpm 9+, Python 3.13+, uv
    AI Claude Opus 4.6 (рекомендация команды)
    Коммуникация Telegram (канал "dumatel alerts")

    Настройка локального окружения¶

    1. Инфраструктура -- dev/local-infra поднимает PostgreSQL 17, Milvus 2.5.14, NATS JetStream, Redis 7, MinIO, etcd: 

    git clone git@gitlab.integra-4d.com:dot.mind/dev/local-infra.git && cd local-infra && docker compose up -d

    2. Backend (Sprut): 

    git clone git@gitlab.integra-4d.com:dot.mind/sprut.git && cd sprut
uv sync && uv run uvicorn source.src.entrypoints.main:app --reload

    3. Frontend (Seadragon): 

    git clone git@gitlab.integra-4d.com:dot.mind/seadragon.git && cd seadragon
pnpm install && pnpm dev

    4. Воркеры (по необходимости): аналогично -- uv sync + uv run python source/src/entrypoints/main.py.

    Карта репозиториев¶

    Группа Репозитории Что делают
    Core seadragon, sprut Фронтенд + API Gateway
    Billing doubloon, anchor, bilge Платежи, карты, подписки
    AI/ML rag.mind/amigofish, rag.mind/anglerfish, krill, sealayers, kraken AI-чат, индексация, OCR, классификация
    Workers descent, gull, shark, dev/convictfish, chronograf Yandex import, email, очистка, планировщик
    Shared shared/orker, shared/sqlalchemy-models-and-migrations, coral Worker framework, DB models, UI components
    DevOps devops/* CI/CD, Helm, Terraform, ArgoCD
    QA diver E2E тесты (Playwright + API)

    Рабочий процесс¶

    Ветки и MR¶

    1. Ветка от main: feature/DOTMIND-XXX-description или fix/DOTMIND-XXX-description
    2. Реализация + unit-тесты
    3. MR с двойным ревью
    4. CI pipeline (lint -> test -> build) должен пройти
    5. После merge -- автосборка Docker + Helm chart -> деплой на dev01

    Внедрение фичи¶

    Формулировка -> проработка функционала -> техреализация -> оценка сроков -> согласование -> спринт (фронту нужен готовый бэк) -> реализация -> unit-тесты -> двойное ревью -> QA -> прод.

    Исправление бага¶

    Сценарий воспроизведения -> написать падающий тест -> исправить -> тест проходит -> ревью.

    Деплой в прод¶

    Только ПН-СР вечером. dev01 стабилизируется под QA. Проверить: NATS-консьюмеры, секреты, миграции. Обязателен план отката. После выкладки: ручной QA + 60 мин мониторинга Sentry.

    Ключевые конвенции¶

    Стек Форматтер Линтер
    Python black (line-length=120) flake8 + mypy + isort
    TypeScript Biome 2.0 Biome + ESLint
    • Package managers: uv (Python), pnpm (TypeScript)
    • Именование сервисов: морская тематика
    • NATS subjects: snake_case (complete_tasks)
    • API routes: kebab-case (/api/billing/buy-credits)

    Полезные команды¶

    # Frontend
pnpm dev                     # Dev server
pnpm generate-api-types      # Типы из OpenAPI Sprut

# Backend
uv run pytest                # Тесты
uv run black . && uv run isort .  # Форматирование

# Infra
docker compose up -d         # local-infra

    Работа при инциденте¶

    Обнаружение (Sentry alert / TG "dumatel alerts" / QA / пользователь) -> SRE реагирует -> уточнить, собрать данные, воспроизвести -> фикс (допустим временный) -> проверка -> восстановление данных пользователей -> post-mortem. 90 мин без решения -- подключать всех.
