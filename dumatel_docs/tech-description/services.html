    Dumatel -- Реестр сервисов¶

    Core¶

    Seadragon (seadragon)¶

    Назначение: Основное фронтенд-приложение платформы -- SPA с AI-стримингом, реализует все пользовательские сценарии. Стек: Next.js 15.5.7, React 18.3.1, TypeScript 5.9, AI SDK 5.0.52, NextAuth 5.0-beta.29, NATS 2.29, Zustand 5.0, TanStack Query 5.90, Tailwind CSS 3.4, Radix UI (shadcn/ui), ProseMirror, CodeMirror 6, PDF.js 5.4, Vitest 3.2, Playwright 1.53, Biome 2.0, pnpm 9.15 Тип: BFF (Backend-for-Frontend) Входы: HTTP от браузера; SSE-стриминг через NATS JetStream Выходы: REST-запросы к Sprut (прокси /api/sprut/); NATS consumer для AI-стримов Зависимости: Sprut (REST), NATS JetStream (AI streaming), Keycloak (через NextAuth OIDC), Sentry Статус: Активная разработка

    Sprut (sprut)¶

    Назначение: Центральный API Gateway -- единая REST-точка входа для фронтенда, оркестратор задач. Стек: Python 3.14, FastAPI 0.128.1, SQLAlchemy (async), asyncpg 0.31, Pydantic 2.12, pymilvus 2.6, aiocache (Redis), aiohttp 3.13, python-jose (JWT), Sentry SDK, Gunicorn 25.0 Тип: API Gateway Входы: REST API (17 роутеров, 60+ эндпоинтов): /api/auth, /api/chat, /api/document, /api/project, /api/organization, /api/billing, /api/profile, /api/attachment, /api/notification, /api/onboarding, /api/yandex, /api/legal, /api/tree, /api/directory, /api/message, /api/health Выходы: NATS Producer (6 subjects: index_tasks, unindex_tasks, complete_tasks, attachment_index_tasks, gull_tasks, descent_tasks); HTTP к Doubloon, Bitrix, UniSender, Yandex; asyncpg к PostgreSQL; pymilvus к Milvus Зависимости: PostgreSQL, Milvus, Redis, NATS, Keycloak, Doubloon, Sentry Статус: Активная разработка

    Doubloon (doubloon)¶

    Назначение: Billing-микросервис -- инкапсулирует бизнес-логику платежей, подписок, карт и кредитов. Стек: Python 3.14, FastAPI 0.122, SQLAlchemy (async), asyncpg 0.31, Pydantic 2.12, aiohttp 3.13, orker 2.1 (NATS Producer), Sentry SDK Тип: Standalone API Входы: REST API (/api/billing -- 17 эндпоинтов): balance, hold, commit, release, buy, change-plan, tariffs, transactions, attach-card, save-card, card management, auto-renew. Авторизация через X-Friend-Token Выходы: NATS Producer (anchor_tasks, anchor_card_tasks); HTTP к Tinkoff T-Kassa Зависимости: PostgreSQL (psdot), NATS, Tinkoff API, Sentry Статус: Активная разработка

    AI/ML Pipeline¶

    Amigofish (rag.mind/amigofish)¶

    Назначение: AI-воркер обработки чат-запросов -- оркестрирует LLM-агента с набором инструментов, стримит ответы через NATS. Стек: Python 3.13, orker, OpenAI SDK, pymilvus, psdot, aiocache (Redis), Jinja2 (промпт-шаблоны), Sentry SDK Тип: Worker (NATS consumer) Входы: NATS subject complete_tasks Выходы: NATS FrontProducer completion.{chat_id} (события: text-delta, reasoning-delta, tool, citations, finish); HTTP к Doubloon (hold/commit), Sealayers (классификация), Sprut (X-Friend-Token) Зависимости: NATS, PostgreSQL, Milvus, Redis, OpenAI/Claude API, Tavily API, Doubloon, Sealayers, Sentry Статус: Активная разработка. LangGraph планируется как замена (amigofish-v2)

    Anglerfish (rag.mind/anglerfish)¶

    Назначение: Воркер индексации документов -- парсит файлы, генерирует embeddings, записывает в Milvus. Стек: Python 3.13, orker, LlamaIndex, OpenAI SDK (embeddings), pymilvus, marker-pdf, psdot, LibreOffice, poppler-utils, Sentry SDK Тип: Worker (NATS consumer, 3 entrypoint) Входы: NATS subjects: index_tasks, unindex_tasks, attachment_index_tasks Выходы: Milvus (write embeddings); NATS FrontProducer (progress events); HTTP к Doubloon (hold для учёта токенов) Зависимости: NATS, PostgreSQL, Milvus, OpenAI API (embeddings), Doubloon, Sentry Статус: Активная разработка. Требует GPU для marker-pdf

    Krill (krill)¶

    Назначение: Воркер парсинга PDF -- OCR и извлечение текста из документов. Стек: Python 3.13, orker, marker-pdf, PyTorch, surya-ocr, spaCy, poppler-utils, LibreOffice Тип: Worker (NATS consumer) Входы: NATS subject nats_parse_subject Выходы: NATS Producer nats_parse_topic_prefix.{document_id} (ParseTaskResult) Зависимости: NATS. Требует GPU Статус: Активный, стабильный

    Sealayers (sealayers)¶

    Назначение: ML-сервис классификации сложности запросов -- предсказывает необходимый уровень reasoning (none/low/medium/high). Стек: Python 3.14, FastAPI 0.128, PyTorch, sentence-transformers (multilingual-e5-base), scikit-learn, HuggingFace Hub, Sentry SDK Тип: Standalone API Входы: POST /api/classify (вызывается из Amigofish по HTTP) Выходы: JSON с уровнем сложности Зависимости: HuggingFace Hub (загрузка моделей), Sentry Статус: Активный, стабильный

    Kraken (kraken)¶

    Назначение: Прокси-сервис к Wolfram Alpha API для математических и вычислительных запросов. Стек: Python 3.11, FastAPI, httpx Тип: Standalone API Входы: POST /evaluate (вызывается из Amigofish) Выходы: HTTP к Wolfram Alpha API Зависимости: Wolfram Alpha API Статус: Активный, ранняя стадия (нет тестов, нет Sentry)

    LangGraph (rag.mind/langgraph)¶

    Назначение: Следующее поколение AI-движка -- multi-agent RAG-система (Coordinator -> classify -> retrieve -> analyze -> format_output). Стек: Python 3.13, LangChain, LangGraph, OpenAI gpt-4o-mini, Tavily, pymilvus, FastAPI, uv Тип: Standalone API (FastAPI) Входы: HTTP API (multi-agent RAG запросы) Выходы: HTTP responses Зависимости: Milvus, OpenAI API, Tavily API Статус: R&D, интеграция в Sprint 23

    Document Processing¶

    Descent (descent)¶

    Назначение: Воркер импорта файлов с Yandex Disk -- скачивает файлы и загружает в Sprut. Стек: Python 3.13, orker, psdot, aiohttp, Sentry SDK Тип: Worker (NATS consumer) Входы: NATS subject nats_descent_subject Выходы: HTTP к Sprut API (загрузка файлов), HTTP к Yandex Disk API; PostgreSQL (нотификации) Зависимости: NATS, Sprut, Yandex Disk API, PostgreSQL, Sentry Статус: Активный

    Billing¶

    Anchor (anchor)¶

    Назначение: Воркер проверки статусов платежей в Tinkoff -- отслеживает успех/неудачу транзакций и привязку карт. Стек: Python 3.13, orker, psdot, aiohttp, aiosmtplib, Sentry SDK Тип: Worker (NATS consumer, 2 entrypoint: main + card) Входы: NATS subjects: nats_anchor_subject, nats_anchor_card_subject Выходы: HTTP к Doubloon (обновление транзакций), HTTP к Tinkoff API (проверка статуса), SMTP (email о результатах) Зависимости: NATS, Doubloon, Tinkoff API, PostgreSQL, SMTP, Sentry Статус: Активная разработка

    Bilge (bilge)¶

    Назначение: Воркер автопродления подписок -- запускается по расписанию Chronograf, вызывает Doubloon. Стек: Python 3.13, orker, aiohttp, aiosmtplib, Sentry SDK Тип: Worker (NATS consumer) Входы: NATS subject nats_bilge_subject (от Chronograf) Выходы: HTTP к Doubloon (автопродление) Зависимости: NATS, Doubloon, SMTP, Sentry Статус: Активный, стабильный

    Infrastructure¶

    Chronograf (chronograf)¶

    Назначение: Планировщик задач -- поллит PostgreSQL каждые 30 секунд, публикует задачи в NATS по расписанию (ONCE, REPEATING, CALENDAR). Стек: Python 3.13, orker (NATS Producer), psdot, Sentry SDK Тип: Scheduler (standalone) Входы: PostgreSQL (таблица schedule) Выходы: NATS Producer -> shark_tasks, bilge_tasks, convictfish_tasks Зависимости: PostgreSQL, NATS, Sentry Статус: Активный, стабильный

    Shark (shark)¶

    Назначение: Почасовой worker системного обслуживания -- 6 операций очистки: таймаут индексации, отмена зависших холдов, таймаут транзакций, удаление истёкших балансов, напоминания о продлении, очистка Yandex-токенов. Стек: Python 3.13, orker, psdot, aiosmtplib, Sentry SDK Тип: Worker (NATS consumer, запуск по расписанию Chronograf) Входы: NATS subject nats_shark_subject Выходы: PostgreSQL (обновление статусов), SMTP (email напоминания) Зависимости: NATS, PostgreSQL, SMTP, Sentry Статус: Активный

    ConvictFish (dev/convictfish)¶

    Назначение: Суточный worker синхронизации данных пользователей с Bitrix CRM. Стек: Python 3.13, orker, psdot, aiohttp-socks, Sentry SDK Тип: Worker (NATS consumer, запуск по расписанию Chronograf) Входы: NATS subject nats_convictfish_subject Выходы: HTTP к Bitrix CRM (обновление записей о пользователях) Зависимости: NATS, PostgreSQL, Bitrix CRM, Sentry Статус: Активный

    Gull (shared/gull)¶

    Назначение: Воркер отправки welcome-email при регистрации и добавления комментариев в Bitrix CRM. Стек: Python 3.13, orker, aiohttp-socks, aiosmtplib, Sentry SDK Тип: Worker (NATS consumer) Входы: NATS subject nats_gull_subject Выходы: SMTP (welcome email), HTTP к Bitrix CRM (webhook) Зависимости: NATS, SMTP, Bitrix CRM, Sentry Статус: Активный

    Clown (clown)¶

    Назначение: Docker Compose для демо и on-premise развёртывания полного стека Dumatel. Стек: Docker Compose, Nginx, PostgreSQL 17, Milvus 2.5, NATS 2, Redis 7, MinIO Тип: Infrastructure Входы: Docker Compose CLI Выходы: Полный стек Dumatel (Sprut, Seadragon, Anglerfish, Amigofish, NATS, PostgreSQL, Milvus, Redis, MinIO) Зависимости: Docker images из cr.selcloud.ru/dotmind/ Статус: Стабильный

    Shared Libraries¶

    orker (shared/orker)¶

    Назначение: Фреймворк для создания NATS JetStream воркеров -- worker lifecycle, retry, dead letter queue, graceful shutdown. Стек: Python 3.14, nats-py, Pydantic, Sentry SDK Тип: Shared library (pip install orker, Nexus registry) Входы/Выходы: API: worker() (main loop), Producer (publish), FrontProducer (publish to frontend), Task (base model) Зависимости: NATS JetStream Статус: Активная разработка. Используется всеми Python-воркерами (10 сервисов)

    psdot (shared/sqlalchemy-models-and-migrations)¶

    Назначение: Единая библиотека SQLAlchemy-моделей, DAL (Data Access Layer) и Alembic-миграций. Стек: Python 3.14, SQLAlchemy 2.x, Alembic, asyncpg, Pydantic Тип: Shared library (pip install psdot, Nexus registry) Входы/Выходы: Модели: User, Organization, Project, Directory, Document, Chat, Message, Attachment, Balance, Tariff, Subscription, Transaction, Card, Freeze, ChargeOff, Notification, Schedule. DAL-классы для CRUD Зависимости: PostgreSQL Статус: Активная разработка. Используется всеми Python-сервисами с БД

    Coral (coral)¶

    Назначение: Монорепозиторий с shared UI-компонентами для фронтенд-приложений. Стек: TypeScript 5, React 19, Tailwind CSS 3.4, Radix UI, Storybook 8.6, Lerna 8, pnpm 10, Biome Тип: Shared library (npm package @coral/ui, Nexus registry) Входы/Выходы: React-компоненты (Button, Input, Dialog, Tooltip и др.), Studio-компоненты Зависимости: Нет серверных зависимостей Статус: Активная разработка (v0.3.12). Вынос Chat-модуля из Seadragon в процессе

    DevOps¶

    devops/workflows¶

    Назначение: Shared CI/CD шаблоны для GitLab CI -- используются всеми репозиториями через include. Стек: GitLab CI YAML Тип: CI/CD templates Шаблоны: .version.yml, .docker.yml, .sentry.yml, .build_chart.yml, .push_chart.yml, .update_umbrella_chart.yml, .deploy_env.yml, .test_env.yml, .verify_pods_state.yml, .npm.yml, .pypi.yml

    devops/deployer¶

    Назначение: Umbrella Helm chart dumatel-system для деплоя всех сервисов как единого релиза. Стек: Helm

    devops/infrastructure¶

    Назначение: IaC -- Terraform для Selectel MKS, Ansible для provisioning (PostgreSQL, NATS, Redis, Milvus, MinIO). Стек: Terraform, Ansible

    devops/environments¶

    Назначение: ArgoCD GitOps-конфигурации для dev/prod namespaces. Стек: ArgoCD Application CRDs

    devops/images¶

    Назначение: Базовые Docker-образы: Python 3.13/3.14, Node 22, infra-tools. Стек: Docker

    devops/devops-infra¶

    Назначение: Kubernetes-манифесты для инфра-компонентов: Redis, NATS, Keycloak, VictoriaMetrics, Fluent Bit, GPU Operator, cert-manager. Стек: Kubernetes YAML
